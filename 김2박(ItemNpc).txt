아이템=========================
- 이벤트에 영향을 줌
- 칼럼1. 이벤트와 아이템을 연결하는 주요키/외래키
- 칼럼2. 아이템 이름
- 칼럼3. 전투용 아이템/ 이벤트용 아이템 -> 인벤 보관용 // 소비용아이템 -> 받자마자 쓰고 가방에 안들어옴
- 칼럼4. 희귀도(교환시 사용)
- 칼럼5.

- 전투용 아이템 테이블 (이름 / 내구도 / 확률)
- 복숭아나무몽둥이 (3회 : 이길확률 40%)
- 덜커덩거리는 망치 (1회 : 이길확률 50%) -> 살인자 업적
- 녹슨 잭나이프 (1회 : 이길확률 60%)
- 주먹 (무제한 <-> 체력반비례 : 이길확률 20%인데 비법 전수받은 후에는 70%)
- 법전 (5회 : 30%)

- 소비용 아이템 테이블 (이름 / 영향주는 스텟영역)
- 육포 hp회복 스텟관여 (+ 히든 개이벤트 해금/안먹고 저녁까지 있으면)
- 붕대 hp회복 스텟관여
- 피로회복제 박카스 정신력 회복 스텟관여
- 가족사진 정신력 회복 스텟관여 (소지하고 다음날 되면 기본 회복수치에 +됨)
- 2인용 침낭 hp회복 (소지하고 다음날 되면 기본 회복수치에 +됨)

- 이벤트용 아이템 테이블 (이름 / 루트코드->이벤트루트)
- 육포 (히든 개이벤트 해금/안먹고 저녁까지 있으면)
- 붕대 (아픈npc와 교류할 수 있게됨)


+ 육포나 붕태는 소비용 아이템이면서 이벤트용 아이템
습득 이벤트 발생시 루트가 아예 갈린다
1. hp가 충분하면 애초에 이벤트용 아이템 타입으로 분류되어서 인벤에 들어가고
2. hp가 부족하면 소비용 아이템으로 분류되어서 바로 사용해 버림
그래서 소비용아이템은 인벤에 들어갈 일이 없다~
무기랑 이벤트아이템만 인벤에 있을 거임


item_id		고유번호

오늘한거
1. item 테이블 작성
2. npc 테이블 작성
3. 상점에서 아이템 교환 알고리즘 작성 중
3-1. 상점에서 구입할 수 있는 아이템 랜덤으로 나타나는 알고리즘 구현
3-2. 아이템 습득시 아이템 테이블에서 기본 값을 가져올 수 있는 메소드 작성
3-3. 아이템 id로 아이템의 정보를 가져올 수 있는 메소드 작성
